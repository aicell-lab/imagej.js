End-to-end: ImageJ.js running macros through the rewriting classloader + worker pool

Boots N workers, preloads CheerpJ into HTTP cache, then uses library-mode to call com.hack.ij.MacroRunner.init(ij.jar + plugin jars) — this constructs our ParallelClassLoader which rewrites every Java class it loads (Thread.start/join, Executors.newXxx, new ThreadPoolExecutor, ForkJoinPool.commonPool, CompletableFuture.supplyAsync). Plugins that use these standard concurrency APIs get parallelized via the Worker pool — no code changes in the plugin jar.
