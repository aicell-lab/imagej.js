CheerpJ Threading Experiment

Goal: verify we can get real parallel execution by hosting CheerpJ inside Web Workers. The WASM/JS analysis confirmed CheerpJ runs single-threaded cooperative fibers — no pthread/SharedArrayBuffer. So real parallelism must come from multiple JVM instances, one per Worker.

Step 3 — Parallelism test


Dispatches N CPU-heavy macros (2048² Gaussian Blur).
If parallelism is real, 4 workers should finish ~4× faster than 4 serial runs on the main thread.
